## [x] Step 1: 불필요한 섹션 및 네비게이션 제거 (MVP 정리)

**오버뷰:**
홈페이지와 Navbar에서 MVP에 불필요한 섹션과 링크를 제거한다. 홈에는 도시 찾기(PopularCitiesSection)만 남기고, Navbar에서는 홈/도시찾기/로그인 관련 링크만 유지한다. Footer도 MVP에 맞게 정리한다.

### 수정/개선 항목

- [x] `components/layout/NavbarClient.tsx`에서 `navLinks` 배열의 "밋업", "커뮤니티" 항목 제거. "홈"(`/`)과 "도시찾기"(`#cities` 또는 별도 페이지)만 유지
- [x] `components/layout/NavbarClient.tsx`에서 검색 버튼(돋보기 아이콘) 제거
- [x] `app/page.tsx` 홈페이지에서 다음 섹션 컴포넌트 import 및 렌더링 제거:
  - [x] `HeroSection` (이메일 수집 CTA)
  - [x] `MediaBadgeSection` (언론 배지)
  - [x] `ValuePropositionSection` (가치 제안)
  - [x] `TrendingCitySection` (트렌딩 도시)
  - [x] `MeetupSection` (밋업)
  - [x] `CommunitySection` (리뷰/Q&A)
  - [x] `StatsSection` (통계)
  - [x] `CityQuizSection` (퀴즈)
  - [x] `FinalCTASection` (최종 CTA)
- [x] `app/page.tsx`에 `PopularCitiesSection`만 남기되, 홈페이지로서 자연스럽게 보이도록 간단한 타이틀/소개 텍스트 추가
- [x] `components/layout/Footer.tsx`에서 MVP에 불필요한 링크(도시비교, 리뷰작성, 밋업, Q&A, 채팅방 등) 제거. 홈/도시찾기/로그인 관련 링크만 유지
- [x] `data/cities.ts`에서 사용하지 않는 데이터 export 제거: `trendingCities`, `meetups`, `recentReviews`, `popularQnAs`, `stats`, `mediaBadges`, `avatars`

### 검증/확인 항목

- [x] 홈페이지(`/`) 접속 시 도시 찾기 섹션만 표시되는지 확인
- [x] Navbar에 "홈", "도시찾기" 링크만 노출되는지 확인
- [x] 로그인 상태에 따라 Navbar에 로그인/회원가입 또는 이메일/로그아웃 버튼이 정상 표시되는지 확인
- [x] Footer에 불필요한 링크가 제거되었는지 확인
- [x] 모바일 햄버거 메뉴에서도 동일하게 정리되었는지 확인
- [x] 빌드 에러 없이 정상 컴파일되는지 확인 (`npm run build`)
- [x] 제거된 섹션 컴포넌트 파일은 삭제하지 않되, 홈에서 렌더링되지 않는지 확인

## [ ] Step 2: 도시 찾기 페이지 분리 및 전체 도시 목록 수정

**오버뷰:**
현재 홈페이지 내 앵커(`#cities`)로 연결되는 도시 찾기를 독립 페이지(`/cities`)로 분리한다. "전체 도시 보기 (40개)" 버튼의 표기를 실제 더미 데이터 개수(10개)에 맞게 수정하고, 도시 목록이 `data/cities.ts`의 데이터만큼만 렌더링되도록 한다.

### 수정/개선 항목

- [ ] `app/cities/page.tsx` 신규 페이지 생성. `PopularCitiesSection` 또는 `CityFilterClient`를 활용하여 도시 목록 표시
- [ ] `components/layout/NavbarClient.tsx`의 "도시찾기" 링크 href를 `#cities`에서 `/cities`로 변경
- [ ] `components/sections/PopularCitiesSection.tsx`에서 "전체 도시 보기 (40개)" 버튼 텍스트를 실제 `cities` 배열 길이에 맞게 동적으로 표시 (예: `전체 도시 보기 (${cities.length}개)`)
- [ ] "전체 도시 보기" 버튼 클릭 시 `/cities` 페이지로 이동하도록 링크 연결
- [ ] `/cities` 페이지에서 `data/cities.ts`의 `cities` 배열 데이터를 전부 표시 (현재 10개)
- [ ] `/cities` 페이지에 기존 카테고리 필터(전체/대도시/해변/산/IT허브)와 정렬 드롭다운 UI 포함
- [ ] 홈페이지(`/`)에서는 도시 일부(예: 상위 4~6개)만 미리보기로 표시하고, 나머지는 `/cities`에서 확인하도록 유도

### 검증/확인 항목

- [ ] `/cities` 페이지 접속 시 10개 도시가 모두 표시되는지 확인
- [ ] Navbar "도시찾기" 클릭 시 `/cities`로 이동하는지 확인
- [ ] "전체 도시 보기" 버튼에 실제 데이터 개수(10개)가 표시되는지 확인
- [ ] "전체 도시 보기" 버튼 클릭 시 `/cities` 페이지로 이동하는지 확인
- [ ] 홈페이지에 도시 미리보기가 적절히 표시되는지 확인
- [ ] 카테고리 필터 및 정렬 드롭다운 UI가 `/cities` 페이지에 렌더링되는지 확인
- [ ] 빌드 에러 없이 정상 컴파일되는지 확인

## [ ] Step 3: 도시 찾기 카테고리 필터 및 정렬 기능 구현

**오버뷰:**
현재 UI만 존재하고 실제 동작하지 않는 카테고리 필터와 정렬 기능을 구현한다. `CityFilterClient.tsx`에서 필터/정렬 상태 변경 시 `data/cities.ts`의 데이터를 실제로 필터링/정렬하여 표시한다.

### 수정/개선 항목

- [ ] `components/sections/CityFilterClient.tsx`에서 카테고리 필터 로직 구현: 선택된 카테고리(`전체`, `대도시`, `해변`, `산`, `IT허브`)에 따라 `cities` 배열을 `category` 필드로 필터링
- [ ] 정렬 로직 구현: 선택된 정렬 기준에 따라 배열 정렬
  - "인기순": `rank` 오름차순
  - "평점순": `rating` 내림차순
  - "생활비 낮은순": `monthlyCost` 오름차순 (문자열에서 숫자 파싱 필요)
  - "최신순": 기본 순서 유지 또는 별도 필드 추가
- [ ] 필터와 정렬이 동시에 적용되도록 구현 (필터 후 정렬)
- [ ] 필터 결과가 0개일 때 "조건에 맞는 도시가 없습니다" 안내 메시지 표시
- [ ] 현재 선택된 필터/정렬 상태가 UI에 시각적으로 표시되도록 확인 (활성 버튼 스타일)

### 검증/확인 항목

- [ ] "대도시" 필터 선택 시 서울, 대구만 표시되는지 확인
- [ ] "해변" 필터 선택 시 부산, 제주, 강릉, 인천, 여수만 표시되는지 확인
- [ ] "산" 필터 선택 시 속초, 평주만 표시되는지 확인
- [ ] "IT허브" 필터 선택 시 대전만 표시되는지 확인
- [ ] "전체" 필터 선택 시 10개 모두 표시되는지 확인
- [ ] "평점순" 정렬 시 rating 높은 순서로 정렬되는지 확인
- [ ] "생활비 낮은순" 정렬 시 비용 적은 순서로 정렬되는지 확인
- [ ] 카테고리 필터 + 정렬 동시 적용이 정상 동작하는지 확인
- [ ] 빌드 에러 없이 정상 컴파일되는지 확인

## [ ] Step 4: 도시명 검색 기능 추가

**오버뷰:**
도시 찾기 페이지(`/cities`)에 도시명 검색 기능을 추가한다. 사용자가 한글 또는 영문 도시명을 입력하면 실시간으로 목록이 필터링된다. 기존 카테고리 필터/정렬과 함께 동작해야 한다.

### 수정/개선 항목

- [ ] `components/sections/CityFilterClient.tsx` 또는 `/cities` 페이지에 검색 입력 필드 추가 (텍스트 Input + 돋보기 아이콘)
- [ ] 검색 상태(state) 추가 및 입력 시 실시간 필터링 구현
- [ ] 한글 도시명(`name` 필드)과 영문 도시명(`nameEn` 필드) 모두에서 검색 가능하도록 구현 (대소문자 무시)
- [ ] 검색어 + 카테고리 필터 + 정렬이 동시에 적용되도록 구현 (검색 → 필터 → 정렬 순서)
- [ ] 검색 결과가 0개일 때 "검색 결과가 없습니다" 안내 메시지 표시
- [ ] 검색어 초기화(X 버튼 또는 입력 필드 클리어) 기능 추가

### 검증/확인 항목

- [ ] "서울" 검색 시 서울만 표시되는지 확인
- [ ] "seoul" 검색 시 서울이 표시되는지 확인 (영문 검색)
- [ ] "부" 검색 시 부산이 포함되어 표시되는지 확인 (부분 검색)
- [ ] "Bu" 검색 시 부산(Busan)이 표시되는지 확인 (영문 부분 검색)
- [ ] 검색어 + 카테고리 필터 조합이 정상 동작하는지 확인 (예: "해변" 필터 + "부" 검색 → 부산만 표시)
- [ ] 검색 결과가 없을 때 안내 메시지가 표시되는지 확인
- [ ] 검색어 초기화 시 전체 목록이 다시 표시되는지 확인
- [ ] 빌드 에러 없이 정상 컴파일되는지 확인

## [ ] Step 5: 도시 상세 보기 페이지 구현

**오버뷰:**
각 도시 카드의 "상세보기" 버튼 클릭 시 이동할 도시 상세 페이지(`/cities/[id]`)를 구현한다. 더미 데이터를 활용하여 도시의 상세 정보, 노마드 관련 지표, 추천 카페/코워킹스페이스 등을 표시한다.

### 수정/개선 항목

- [ ] `app/cities/[id]/page.tsx` 동적 라우트 페이지 생성
- [ ] `data/cities.ts`에 도시 상세 더미 데이터 추가. 각 도시별로 다음 항목 포함:
  - [ ] 도시 소개 텍스트 (2~3문장)
  - [ ] 장점/단점 목록 (각 3개씩)
  - [ ] 추천 카페/코워킹스페이스 (각 2~3개, 이름/위치/특징)
  - [ ] 월별 기후 정보 (간단한 텍스트)
  - [ ] 교통 정보 (간단한 텍스트)
  - [ ] 기존 데이터 활용: 생활비, 인터넷 속도, 추천률, 월세, 카페 밀도
- [ ] 상세 페이지 레이아웃 구현:
  - [ ] 상단: 도시명(한글/영문), 카테고리 배지, 평점, 추천률
  - [ ] 핵심 지표 카드 영역: 월 생활비, 인터넷 속도, 월세, 카페 밀도
  - [ ] 도시 소개 섹션
  - [ ] 장점/단점 섹션
  - [ ] 추천 장소 섹션 (카페/코워킹스페이스)
  - [ ] 기후 및 교통 정보 섹션
- [ ] 도시 카드의 "상세보기" 버튼에 `/cities/[id]` 링크 연결 (`PopularCitiesSection.tsx` 또는 `CityFilterClient.tsx` 수정)
- [ ] 존재하지 않는 도시 ID 접근 시 404 또는 적절한 안내 페이지 표시
- [ ] 상세 페이지에서 도시 목록(`/cities`)으로 돌아가는 "← 목록으로" 버튼 추가

### 검증/확인 항목

- [ ] 도시 카드 "상세보기" 클릭 시 해당 도시 상세 페이지로 이동하는지 확인
- [ ] 10개 도시 각각의 상세 페이지(`/cities/1` ~ `/cities/10`)가 정상 렌더링되는지 확인
- [ ] 상세 페이지에 더미 데이터(소개, 장단점, 추천 장소 등)가 모두 표시되는지 확인
- [ ] 핵심 지표(생활비, 인터넷 속도 등)가 기존 데이터와 일치하는지 확인
- [ ] 존재하지 않는 ID(예: `/cities/99`) 접근 시 적절한 처리가 되는지 확인
- [ ] "← 목록으로" 버튼 클릭 시 `/cities`로 이동하는지 확인
- [ ] 모바일/태블릿 반응형 레이아웃이 정상인지 확인
- [ ] 빌드 에러 없이 정상 컴파일되는지 확인
